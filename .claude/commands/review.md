# 기획 검토 (Review)

기획서와 업무 리스트를 대조하여 필요한 부분을 업데이트합니다.

## 실행 순서

### 1. 파일 읽기
- `docs/planning.md` 읽기
- `docs/tasklist.md` 읽기

### 2. 대조 및 검토
현재 대화 맥락(사용자가 말한 변경사항, 새 기획 등)과 두 파일을 대조하여 확인:
- 기획서에 반영되지 않은 결정사항이 있는가
- 업무 리스트에서 완료 처리가 안 된 항목이 있는가
- 새로 추가해야 할 태스크가 있는가
- 순서나 Phase 조정이 필요한가

### 3. 업데이트
변경이 필요한 경우:
- `docs/planning.md` 업데이트 (업데이트 이력 날짜 포함)
- `docs/tasklist.md` 업데이트 (완료 항목 체크, 신규 항목 추가 등)

### 4. 변경 요약
업데이트한 내용을 사용자에게 간단히 요약하여 보고.
변경사항이 없으면 "기획서와 업무 리스트가 최신 상태입니다"라고 알림.

## 주의사항
- 커밋/푸시는 하지 않음 (사용자가 별도 요청 시 진행)
- 사용자가 명시한 내용 외에 임의로 기획을 추가하거나 삭제하지 않음
