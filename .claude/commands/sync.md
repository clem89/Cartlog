# 동기화 (Sync)

오늘의 작업을 커밋, 푸시하고 작업 내용을 docs에 기록합니다.

## 실행 순서

### 1. 변경사항 파악
다음 명령어로 현재 상태를 확인합니다:
- `git status` — 변경된 파일 목록
- `git diff` — 변경 내용 상세
- `git log --oneline -10` — 최근 커밋 이력

### 2. 커밋
- 변경된 파일을 분석하여 한국어로 명확한 커밋 메시지 작성
- `git add` 로 관련 파일 스테이징 (민감 파일 제외)
- `git commit` 으로 커밋 생성
- Co-Authored-By 태그 포함

### 3. 푸시
- `git push origin master` (또는 현재 브랜치) 로 원격에 푸시

### 4. 작업 일지 저장
- `docs/` 폴더가 없으면 생성
- 오늘 날짜 기준 파일명: `docs/YYYY-MM-DD.md`
  - 이미 파일이 존재하면 내용을 추가(append)
- 마크다운 형식으로 아래 내용 포함:
  ```
  ## HH:MM — [작업 제목]

  ### 변경 파일
  - 변경된 주요 파일 목록

  ### 작업 내용
  - 구체적으로 무엇을 했는지 bullet point로 정리
  - 추가/수정/삭제된 기능, 버그 수정 등

  ### 커밋
  `커밋 해시` — 커밋 메시지
  ```
- 작업 일지 파일도 git add → commit → push 진행

## 주의사항
- `.claude/`, `.env`, 인증 정보 등 민감 파일은 절대 커밋하지 않음
- 변경사항이 없으면 커밋 없이 사용자에게 알림
- 작업 일지는 항상 한국어로 작성
